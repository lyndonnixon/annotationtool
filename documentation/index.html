ConnectME Annotation Tool

The ConnectME annotation tool is based on PHP classes, PHP script files, JSON language files and jQuery/JavaScript files. This documentation describes how they are used and how they interact with each other.

Please remember that you need a list of external libraries and plugins and a working instance of the ConnectME Framework to use this sources. Follow this instructions to set up your infrastrucure correctly.

Core

The core files provide the main functions of the annotation tool. Those are logging in using an OpenID URL, loading a video from the CMF, annotating a video, logging out and getting help by screencasts.

Administration

The administration file allows selecting CMF instances, granting OpenID users administration privileges and viewing log files. The annotation tool logs performed operations (logging in, logging out, loading videos) and every interaction with the selected CMF instance (adding annotations, updating annotations and deleting annotations).

Plugins

The annotation tool allows adding search plugins which are used for adding linked data search sources (like DBPedia). It is also possible to add non linked-data search sources (which has been done when implementing the Goole Maps search plugin).

Every plugin needs to get added to the settings file and needs to follow a defined structure. Follow this instructions to create your own search plugin:

  • Create a directory with the name of your plugin in the plugins directory (e.g. /plugins/my-search-plugin/)

  • The directory name has to be the name of your plugin and needs to get added to the settings file. Extend the property $resourceplugins with the settings of your plugin.

    public $resourceplugins =
  array(
    array(
      'name' => 'my-search-plugin',
      'url' => 'my-plugin-base.url',
      'loader' => 'plugins/my-search-plugin/my-search-plugin.data-lookup.php',
      'parameters' => array(
        'http://my-plugin-base.url/ontology#label',
        'http://my-plugin-base.url/ontology#abstract'
      )
    )
  );

    Your plugin must be able to handle a search query and also be able to search for results for a specified result identifier (e.g. http://dbpedia.org/resource/Jim_Parsons)

  • Place index.php in your plugin directory and implement the search function. It must be able to handle a request like: POST /plugins/my-search-plugin/ "query=my query"

    Your search file has to produce a JSON string of the follwing structure

    {
  "results":[
    {
      "label": "My first result",
      "uri": "http://my-plugin-base.url/resource/my-first-result",
      "description": "This is the description of my first search result."
    },
    ...
  ]
}

  • Place the file my-search-plugin.data-lookup.php in your plugin directory. This file has to be able to handle requests like: POST /plugins/my-search-plugin/my-search-plugin.data-lookup.php "uri=http://my-plugin-base.url/resource/my-first-result" and produce a JSON response which follows this structure:

    {
  "response": {
    "http://my-plugin-base.url/ontology#label": "My first result",
    "http://my-plugin-base.url/ontology#abstract": "This is the description of my first search result."
  }
}

    It is required that the property names of the response are exactly the same, which have been specified in the settings in $resourceplugins! Otherwise loading labels and descriptions won't be possible!

  • Finally you have to create a (several) language file(s) for your plugin. The language file has to get placed in the lang directory (plugins/my-search-plugin/lang/) and its name consists of lang.[ISO 639-1 language code].json (e.g. lang.en.json).

    {
  "strings": {
	  "NAME": "Tell the user what your plugin is searching for"
  }
}

  • If you followed the instructions properly, your search plugin should aready be available in the annotation tool.
